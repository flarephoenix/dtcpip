# 日本の著作権法と DTCP-IP

> 免責: 本ドキュメントは法的助言ではない。具体的な判断は弁護士に相談すること。

## 関連条文の概要

### 第30条（私的使用のための複製）

> 著作権の目的となっている著作物（以下「著作物」という。）は、個人的に又は家庭内その他これに準ずる限られた範囲内において使用すること（以下「私的使用」という。）を目的とする場合には、（中略）その使用する者が複製することができる。

**→ 個人・家庭内での複製は原則 OK**

### 第30条第2項（技術的保護手段の回避による複製の禁止）

> 技術的保護手段の回避（（中略）技術的保護手段に用いられている信号の除去又は改変（（中略）を行うことにより、当該技術的保護手段によって防止される行為を可能とすること）をいう（（中略）））により可能となり、又はその結果に障害が生じないようになった複製を、その事実を知りながら行う場合には、適用しない。

**→ 技術的保護手段を「回避」して複製することは、私的使用であっても違法**

2012年10月1日の法改正により、私的複製であっても技術的保護手段の回避による複製は禁止された（民事上違法）。

## DTCP-IP は「技術的保護手段」か？

**→ はい、該当すると考えられる。**

- DTCP-IP はコンテンツを AES-128 で暗号化し、認証された機器以外での復号を防ぐ技術
- この「暗号化によるアクセス制限」が著作権法上の「技術的保護手段」に該当
- DVD の CSS、Blu-ray の AACS と同様の扱いと解釈される

参考: 2012年の法改正でDVDリッピング（CSS回避）が明示的に禁止されたが、DTCP-IP の扱いはさらに論点が残る。

## 「回避」と「実装」の違い（グレーゾーン）

**重要な論点**: DTCP-IP の「回避」と「実装」は異なる可能性がある。

| 行為 | 法的評価 |
|---|---|
| DTCP-IP 保護を「破る」（暗号鍵の不正取得等） | 明確に違法 |
| DTCP-IP プロトコルを仕様通りに「実装」する（DTLA 証明書なし） | **グレーゾーン** |
| DTLA ライセンス済みの実装（DiXiM 等）を利用する | 合法 |

「プロトコルの実装」が「技術的保護手段の回避」にあたるかどうかは、証明書なしに認証を突破する行為を含むかどうかによる。DTLA 証明書なしに AKE を完了させることは、実質的に保護手段を回避していると解釈される可能性が高い。

## 裁判例・行政指導

**DTCP-IP に特化した裁判例は確認できていない（2026年3月時点）。**

関連する動向:
- DVD リッピングの違法化（2012年）: CSS 回避が私的使用例外の対象外と明示
- その後の解釈により、録画機器の DRM 回避全般に同様の論理が適用される方向

## ダビング10 の法的根拠

「ダビング10」は法律ではなく、放送業界・権利者・メーカー間の**任意の取り決め**。

- 2008年6月2日導入
- コピーワンスから「9回コピー＋1回ムーブ」へ緩和
- 法的根拠は著作権法ではなく、業界合意

## 実際のリスク評価（個人利用の場合）

| シナリオ | 法的リスク | 現実的リスク |
|---|---|---|
| DiXiM 等の正規ライセンス済みソフトを使う | なし | なし |
| 自作アプリ（DTLA 証明書なし）で BDZ から保存 | 民事上違法の可能性 | 低い（非公開・非配布前提） |
| 自作アプリを配布する | 違法の可能性 + 刑事リスク | 高い |
| DTCP-IP 回避ツールを販売・商用提供 | 刑事罰対象（3年以下懲役・300万円以下罰金） | 高い |

**個人利用・非配布前提で自作する場合**: 法的には民事上の違法となる可能性はあるが、刑事罰の対象にはなりにくく、現実的なリスクは低い。ただし法的にクリアとは言い切れない。
